# Pipeline Overview

A detailed explanation of the VGP assembly pipeline — what each phase does and why.

---

## Why This Pipeline?

The VGP (Vertebrate Genomes Project) pipeline is the gold standard for producing near-perfect genome assemblies. It was developed by an international consortium to generate reference genomes for all vertebrate species. We use it here for *S. cerevisiae* because:

1. It produces **chromosome-level, haplotype-phased** assemblies
2. It is **reproducible** — every step is documented and runs on Galaxy
3. It integrates **three data types** that complement each other
4. It includes **built-in quality checkpoints** at every stage

---

## Phase 1 — Prepare the Data

### Why clean the reads first?

PacBio HiFi reads are generated by circularizing DNA and sequencing around the circle multiple times. The circular adapters can occasionally end up inside the read itself. These contaminated reads would cause errors during assembly — the assembler would see artificial sequences that don't exist in the genome.

**Cutadapt** removes entire reads that contain adapters. We use strict settings (min overlap=35 bp) to avoid false positives while catching all genuine contamination.

### Why count k-mers before assembly?

K-mer analysis with **Meryl** and **GenomeScope2** gives us a "genome preview" before we even start assembling:
- **Genome size** — so we know what to expect
- **Heterozygosity** — tells us how different the two chromosome copies are
- **Repeat content** — warns us about difficult regions
- **Coverage depth** — confirms our sequencing depth is sufficient
- **Error rate** — confirms read quality

This information is used to set parameters for downstream tools (like the expected genome size in Gfastats).

---

## Phase 2 — Assemble the Genome

### How Hifiasm works

Hifiasm is specifically designed for PacBio HiFi reads. It works in three stages:

1. **Error correction** — 3 rounds of haplotype-aware correction that fix sequencing errors while preserving genuine heterozygous variants
2. **Assembly graph** — builds a string graph where heterozygous regions form "bubbles"
3. **Phasing** — Hi-C contact data resolves each bubble, assigning contigs to Hap1 or Hap2

Without Hi-C data (solo mode), hifiasm produces a primary + alternate pseudohaplotype assembly with switch errors. **Hi-C phased mode** produces two fully phased haplotypes with far fewer errors.

### Why do we get GFA files?

GFA (Graphical Fragment Assembly) stores both sequences AND their overlap relationships. It contains more information than FASTA but most downstream tools expect FASTA. **Gfastats** converts GFA to FASTA while preserving all sequence information.

## Phase 3 — Quality Control

### BUSCO — functional completeness

BUSCO checks whether the assembly contains all the genes that *should* be there. For yeast, 2,137 genes are expected to be present at exactly one copy per haplotype. High completeness (>95%) and low duplication (<5%) confirm:
- The assembly is not missing important genomic regions
- There are no false duplications (regions assembled twice)

### Merqury — sequence-level completeness

Merqury works at the sequence level rather than gene level. It checks whether every k-mer (31-base substring) found in the reads is also found in the assembly. This catches:
- Missing genomic regions (k-mers in reads but not in assembly)
- Assembly errors (unexpected k-mer counts)
- Phasing quality (are the two haplotypes well-separated?)

Together, BUSCO and Merqury provide complementary views of assembly quality.

---

## Phase 4 — Scaffold to Chromosome Level

After assembly, we have **contigs** — gapless sequences — but they are not yet in chromosome order. Scaffolding connects contigs into **scaffolds** using long-range information.

### Bionano optical maps

Bionano technology images individual DNA molecules in nanochannel arrays. Restriction enzymes cut at specific sequences, and the resulting label pattern is unique for each genomic region. These physical maps can span hundreds of Mb — far beyond what any sequencing technology can cover.

The Bionano Hybrid Scaffold tool aligns sequence contigs to physical maps to:
- Order contigs along the chromosome
- Orient contigs in the correct direction
- Estimate gap sizes between contigs

### Hi-C scaffolding with YaHS

Hi-C captures 3D chromosome contacts in the nucleus. The key biological principle: **DNA regions on the same chromosome interact far more frequently than regions on different chromosomes**.

YaHS uses this to:
1. Calculate "joining scores" for every possible pair of contig ends
2. Build a scaffolding graph where high-scoring pairs are connected
3. Traverse the graph to assemble chromosome-scale scaffolds

YaHS is used after Bionano because it can resolve connections that optical maps cannot (e.g., repetitive regions) and provides the final chromosome-level organization.

### Why map Hi-C reads separately?

Standard paired-end mapping assumes a known insert size distribution. Hi-C ligation products can be any size from 1 bp to hundreds of Mb — there is no "expected" insert size. Mapping each read independently avoids this problem and ensures correct alignment.

## Quality Checkpoints Summary

```
After Step 3  (GenomeScope2):  → Verify genome size and heterozygosity match expectations
After Step 5  (Gfastats):      → Check contig count and N50
After Step 6  (BUSCO):         → Confirm >95% gene completeness, <5% duplication
After Step 7  (Merqury):       → Confirm >99% k-mer completeness, good phasing
After Step 11 (Initial map):   → Confirm contigs are internally correct
After Step 14 (Final map):     → Confirm chromosome-level scaffolding succeeded
```
